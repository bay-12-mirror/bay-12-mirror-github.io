 This is the SEC_SELECT binding in interface.txt, which you can edit in the esc -> key bindings menu ("secondary select" under "general").

I think there was a misunderstanding here. Loci, as far as I can tell, this isn't strictly a duplicate. 0003007 may be hard/impossible for Toady to fix, but this report isn't asking for "shift+enter" support under ncurses - it's asking for a way to rebind that key to work around 0003007, which there already is. Sure, SEC_SELECT is used in places besides the military screen, but it'll have the same issue under ncurses in all of those places.

Huelter - I'm pretty sure I found the screen you were talking about (m-s-e), but if that's not it, only two other bindings use Shift+Enter by default, so try changing these too:
- SELECT_ALL, under key bindings -> "general" -> "select all"
- D_MILITARY_ALERTS_SET_RETAIN, under key bindings -> "Militia" -> "Main: Military, Alerts, Set/Retain"
